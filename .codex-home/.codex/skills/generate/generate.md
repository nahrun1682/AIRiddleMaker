# なぞなぞ生成スキル

以下の制約に従って日本語なぞなぞを **1問だけ** 生成してください。

## 生成プロセス（必ず順番に実行）

1. まず **答え** をひとつ決める（身近な具体物・日常現象に限る）
2. その答えの特徴から **逆説（AなのにB / AするほどB）** を一つだけ見出す
3. 逆説を問題文に仕立てる（短く・余計な説明を入れない）
4. 自己検証：問題文だけ読んで答えが一つに定まるか確認する（他の答えでも成立しないか？）
5. 自己検証：答えを聞いた瞬間「なるほど！」と思えるか確認する
6. 自己検証：答えに直結するキーワードが問題文にないか確認する
7. 自己検証：問題文に「虫」「魚」「鳥」等の分類語を含めていないか？含めると候補が絞れすぎてクイズになる

## 発想のコツ（定番を避けるために）

- **動作の逆転** を探す: その物の「普通の使い方」と「実際の効果」のギャップ
- **名前と実態のズレ** を探す: 名前から想像する姿と実際の違い
- **数え方・量の逆転** を探す: 増えるほど減る、大きいほど軽い、など
- **最初にメジャーな答えが浮かんだら捨てる**: 定番こそが最大の敵
- **テーマが指定された場合でも、マイナーな対象を選ぶ**: 犬・猫・象ではなくモグラ・ナマケモノ・ダンゴムシなど
- **「〜する動物は？」「〜な鳥は？」パターンは使わない**: 特徴当てクイズになりやすい
- **主語を隠す**: 問題文の主語を「それ」「こいつ」すら使わず、現象だけで問う
- **分類語を問題文に入れない**: 「〜な虫は？」「〜な魚は？」と書くと候補が即座に絞れてしまう。「〜なのはなに？」にする
- **「その答えでしか成立しない」特徴を使う**: 犬が吠えるのは普通すぎる。モグラが土を掘る→地面にモグラ塚ができるのは唯一的

## 目指すべき品質の参考例

良いなぞなぞはこのような構造を持つ：
- 「切っても切っても増えるものは？」→「トランプ（カード）」 … 切るの二重意味
- 「上がらないと使えないのに、上がると使えなくなるものは？」→「花火」 … 二方向逆説
- 「頭を叩かれるほど、いい仕事をするものは？」→「釘」 … 痛みと成果の逆転

※ これらはあくまで構造の参考。同じ答えを使ってはいけない。
## 必須条件

- 答えが一つに定まること（複数解は不可）
- 一つの現象から一つの逆説で成立すること（複数ステップ禁止）
- 逆説は二方向で成立すること
- 答えを聞いた瞬間に「なるほど！」と納得できること
- 子供の知識でも答えを聞けば容易に理解できるもの

## 作成指針

- 抽象概念（秘密・時間・平和など）から作らない
- 日常の物理現象や身近な事実を観察して作る
- 問題文は **1〜2文（40字以内推奨）** に収める
- 答えをほのめかす単語を問題文に入れない

## 禁止答えリスト（これらを答えにしてはいけない）

影、時間、風、沈黙、炎、記憶、夢、鏡、氷、秘密、平和、空気、ヤドカリ、カメレオン、食欲、つらら、カタツムリ、キリン、ハムスター、コウモリ

## 禁止パターン

- 「私は何でしょう」「What am I?」の定番フォーム
- 英語発想の直訳
- 答えが複数成立する問題
- 単純な知識を問うクイズ（特定の生物の生態知識がないと解けない等）
- 答えに直結するキーワードを問題文に含める（答えが容易に推測できてしまう）

## 日本語固有パターン

以下のいずれかを使うこと。

- `paradox`: 逆説・現象系
- `pun`: 同音異義・ダジャレ系
- `char_extract`: 文字分解系
- `reverse_read`: 逆読み系
- `kanji_structure`: 漢字の部首・構造系

## 出力フォーマット（JSONのみ）

以下のJSONのみを返すこと。他のテキストは一切禁止。

```json
{"question":"問題文","answer":"答え","pattern":"パターン名"}
```
