# なぞなぞ生成スキル

以下の制約に従って日本語なぞなぞを **1問だけ** 生成してください。

---

## Step 0: パターン選択

以下の5パターンからランダムに選ぶ。**パターンが指定されている場合はそのパターンに固定する。**
一度選んだパターンは、scorer から構造的限界を指摘されない限り変えずに粘ること。

| パターン | 特徴 | 起点 |
|---|---|---|
| `paradox` | 逆説・現象の矛盾 | 答えから逆説を発見 |
| `pun` | 同音異義・ダジャレ | 音の響きから二重意味を発見 |
| `char_extract` | ひらがな・カタカナの文字操作 | 文字列の分解・抽出から発見 |
| `reverse_read` | 逆から読むと別の意味 | 逆読みで別の言葉になる語を発見 |
| `kanji_structure` | 漢字の部首・偏旁の構造 | 漢字の視覚的構造から発見 |

---

## パターン別 思考ルート

選んだパターンに対応するルートを **必ず順番に** 実行すること。

### ルート A: `paradox`（答え起点 → 逆説発見）

**〈発散〉**
1. 身近な具体物・日常現象から **答え候補を3つ** 挙げる
2. 各候補について「AなのにB」「AするほどB」の逆説を探す
3. 最もマイナーで意外性のある候補を1つ選ぶ（最初に浮かんだものは捨てる）

**〈構築〉**
4. 選んだ逆説を短い問題文に仕立てる（現象だけで問い、主語を隠す）
5. 問題文は1〜2文・40字以内

**〈検証〉**
6. 問題文だけ読んで答えが一つに定まるか？（他の答えでも成立しないか検証）
7. 答えを聞いた瞬間「なるほど！」と思えるか？

**参考例（構造のみ参考、同じ答えを使うな）:**
- 「頭を叩かれるほど、いい仕事をするものは？」→「釘」 … 痛みと成果の逆転

**コツ:**
- 動作の逆転、名前と実態のズレ、数え方の逆転を探す
- 「その答えでしか成立しない」特徴を使う

---

### ルート B: `pun`（音韻起点 → 二重意味の発見）

**〈発散〉**
1. 答え候補の語を1つ選び、その **読み（ひらがな）** を書き出す
2. その読みと同じ音・近い音を持つ **別の意味の言葉を3つ以上** 列挙する
   - 同音異義語（例: はし → 橋/箸/端）
   - 音の一部が重なる語（例: さる → 去る/猿）
   - 複合語の切り方違い（例: ぱんだ → パンだ/パンダ）
3. 最も意外な組み合わせを選ぶ

**〈構築〉**
4. 二つの意味を自然に繋ぐ問題文を作る
5. 一方の意味にミスリードし、もう一方が答えになる構造にする
6. 問題文は1〜2文・40字以内

**〈検証〉**
7. ダジャレとして成立するか？（音のズレが大きすぎないか）
8. 答えを聞いて「うまい！」と思えるか？（単なる同音置換だけでは不十分）

**参考例（構造のみ参考、同じ答えを使うな）:**
- 「パンはパンでも食べられないパンは？」→「フライパン」 … 複合語内のパン

**コツ:**
- 音読み・訓読みの両方を検討する（読みが増えるほど言葉遊びの幅が広がる）
- 慣用句や日常表現に潜む同音異義に注目する
- 単純な「〜は〜でも」の定型に頼りすぎない

---

### ルート C: `char_extract`（文字起点 → 操作ギミックの発見）

**〈発散〉**
1. 答え候補の語をひらがな/カタカナで書き出す
2. その文字列に対する操作を複数試す:
   - 特定の文字を抜くと別の語になる（例: 「しりとり」→「り」を取る→「しと」？）
   - 文字を入れ替えると別の語になる
   - 文字を1つ足すと別の語になる
   - 繰り返すと別の語になる
3. 面白い操作が見つかった候補を選ぶ

**〈構築〉**
4. 文字操作をヒントとして暗示する問題文を作る（操作自体を直接言わない）
5. 問題文は1〜2文・40字以内

**〈検証〉**
6. 文字操作が正確か？（一文字もずれていないか再確認）
7. 問題文から操作と答えに論理的にたどり着けるか？

**参考例（構造のみ参考、同じ答えを使うな）:**
- 「真ん中の文字を抜くと怒り出す食べ物は？」→「パセリ」（セを抜く→パリ≒怒りのパリッ？）

**コツ:**
- ひらがなとカタカナの両方で試す（表記変換で別の語が見える場合がある）
- **操作結果が実在する日本語であること** を必ず確認

---

### ルート D: `reverse_read`（逆読起点 → 意味の発見）

**〈発散〉**
1. 日常語を片っ端から逆さに読んで、**別の意味のある語になるもの** を探す
   - 例: しんぶん→ンブンシ（×） / いるか→かるい（○）/ ねこ→こね（○）
2. 逆読みで意味が通る候補を3つ以上集める
3. 両方の意味を使って面白い問題が作れるものを選ぶ

**〈構築〉**
4. 順読みの意味と逆読みの意味を対比・接続する問題文を作る
5. 問題文に「逆さ」「反対」「ひっくり返す」等のヒントを自然に組み込む
6. 問題文は1〜2文・40字以内

**〈検証〉**
7. 逆読みが本当に正確か？（文字単位でひっくり返して確認）
8. 問題文のヒントだけで逆読みだと気づけるか？

**参考例（構造のみ参考、同じ答えを使うな）:**
- 「ひっくり返すと軽くなる海の生き物は？」→「イルカ」（イルカ→カルイ=軽い）

**コツ:**
- 3文字〜4文字の語が逆読みしやすい
- 逆読み結果が日常語であるほど「なるほど感」が高い

---

### ルート E: `kanji_structure`（構造起点 → 漢字の視覚的な謎かけ）

**〈発散〉**
1. 面白い構造を持つ漢字を候補として3つ挙げる
2. 各候補の **部首・偏旁・構成要素** を書き出す
   - 例: 「森」= 木 + 木 + 木、「想」= 相(木+目) + 心、「岩」= 山 + 石
3. 構成要素の組み合わせが意外な意味を持つものを選ぶ

**〈構築〉**
4. 漢字の構造を言葉で表現する問題文を作る
   - 例: 「木が3本集まってできるものは？」→「森」
5. ただし上記のように直接すぎるのはNG。少しひねりを加える
6. 問題文は1〜2文・40字以内

**〈検証〉**
7. 漢字の分解が事実として正確か？（部首辞典レベルで正しいか確認）
8. 問題文から答えの漢字にたどり着けるか？

**参考例（構造のみ参考、同じ答えを使うな）:**
- 「田んぼに力を入れると何になる？」→「男」（田 + 力 = 男）

**コツ:**
- 常用漢字から選ぶ（画数が多すぎると分解が複雑になりすぎる）
- 構成要素がそれぞれ独立した意味を持つ漢字が使いやすい
- **漢字の分解は自分の知識を過信しない**: 不確実なら避ける

## 共通検証（全パターン共通、構築後に必ず実行）

- [ ] 答えに直結するキーワードが問題文にないか？
- [ ] 答えが一つに定まるか？（他にもっともらしい答えがないか？）
- [ ] 問題文に「虫」「魚」「鳥」等の分類語を含めていないか？
- [ ] 答えを聞いた瞬間に「なるほど！」と思えるか？
- [ ] 問題文は40字以内に収まっているか？
- [ ] 子供の知識でも答えを聞けば理解できるか？
- [ ] 禁止答えリストに含まれていないか？

## 作成指針

- 抽象概念（秘密・時間・平和など）から作らない
- 日常の物理現象や身近な事実を観察して作る
- 問題文は **1〜2文（40字以内推奨）** に収める
- 答えをほのめかす単語を問題文に入れない
- **「〜する動物は？」「〜な鳥は？」パターンは使わない**: 特徴当てクイズになりやすい
- **主語を隠す**: 問題文の主語を「それ」「こいつ」すら使わず、現象だけで問う
- **分類語を問題文に入れない**: 「〜な虫は？」「〜な魚は？」→「〜なのはなに？」にする
- **最初にメジャーな答えが浮かんだら捨てる**: 定番こそが最大の敵

## 禁止答えリスト（これらを答えにしてはいけない）

影、時間、風、沈黙、炎、記憶、夢、鏡、氷、秘密、平和、空気、ヤドカリ、カメレオン、食欲、つらら、カタツムリ、キリン、ハムスター、コウモリ

## 禁止パターン

- 「私は何でしょう」「What am I?」の定番フォーム
- 英語発想の直訳
- 答えが複数成立する問題
- 単純な知識を問うクイズ（特定の生物の生態知識がないと解けない等）
- 答えに直結するキーワードを問題文に含める（答えが容易に推測できてしまう）

## 出力フォーマット（JSONのみ）

以下のJSONのみを返すこと。他のテキストは一切禁止。

```json
{"question":"問題文","answer":"答え","pattern":"パターン名"}
```
