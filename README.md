# 🎌 AIRiddleMaker
**日本語特化なぞなぞ生成システム**

## Overview

日本語なぞなぞをAIで生成するシステム。AOF論文（arXiv:2508.18709）の手法を参考に、日本語向けに再設計したオリジナリティフィルタリングを実装する。

---

## Motivation

AOF論文は日本語で実験を行ったが以下の問題があった。

- フィルタリングの参照セットが英中データ（BiRdQA）であり、日本語なぞなぞを英中データで類似度判定するという構造的欠陥がある
- 著者陣（Algoverse AI Research）に日本語ネイティブが存在しない
- 論文が生成した日本語なぞなぞ「屋根にはいるのに、家にいないものは何？」は答えが「ね」「や」など複数成立してしまい一意性がない
- プロンプトが禁止リスト頼みで、日本語固有の語呂・漢字構造・音遊びへの言及が皆無

本システムはこれらの問題を解決する。

---

## AOF（Adaptive Originality Filtering）とは

本システムはAOF論文の手法を参考に日本語向けに再設計している。まず原手法を説明する。

AOFは3つのコンポーネントで構成される。

**① プロンプト制約**
定番フォームや頻出答えを禁止し、文法バリエーションを強制する。ただし禁止リスト頼みであり日本語固有の構造には対応していない。

**② セマンティックフィルタリング**
生成したなぞなぞを `all-MiniLM-L6-v2` でベクトル化し、参照データセット（BiRdQA）内の既存なぞなぞ全件とコサイン類似度を計算。閾値θ=0.75未満なら採択、以上なら既存のなぞなぞに似すぎとしてボツ。

**③ リジェクションサンプリングループ**
フィルターを通過するまで生成→判定を繰り返す。リトライ上限に達したら打ち切り。

```
プロンプト制約付きで生成
        ↓
MiniLMでベクトル化
        ↓
参照セットとコサイン類似度計算
        ↓
< 0.75 → 採択
≥ 0.75 → ボツ → 再生成
```

## 本システムのアプローチ：ネット検索による代替

AOFの類似性フィルタ（MiniLM＋コサイン類似度＋BiRdQA）を **ウェブ検索サブエージェント** に置き換える。

### なぜネット検索の方が優れているか

日本語なぞなぞはウェブ上に数万件単位で存在する。静的データセット数千件と比較して母数が圧倒的に大きく「既存のなぞなぞに似ているか」の判定精度が高い。また多言語モデルで日本語を無理やり処理する必要もない。

実際に本プロジェクトで作成したなぞなぞをウェブ検索で確認した結果、以下がネット未検出であることを確認済み。

- 「食べるほど減るのに、食べないと増える → 食欲」（オリジナリティ 8.5/10）
- 「開けたことないのに、誰でも知ってる中身 → 卵」（オリジナリティ 9.2/10）

## System Architecture

```
Codex CLI（リーダーエージェント）
        │
        ├─ なぞなぞ生成（プロンプト制約付き）
        │
        ├─ ウェブ検索サブエージェント（MCP）
        │       └─ 生成したなぞなぞをそのまま検索
        │           ヒット    → ボツ → 再生成
        │           ヒットなし → 採択
        │
        └─ 出力
```

## 日本語なぞなぞ生成のプロンプト設計

AOF論文のプロンプトは禁止リスト頼みだった。本システムでは以下の知見に基づきプロンプトを設計する。

### 良いなぞなぞの条件

**① 一現象・一逆説・一答えの法則**
複数ステップの論理は弱い。一つの現象から一つの逆説が生まれ、答えが一意に定まること。

> ❌ 「濡れるほど軽くなる → 雑巾」
> 「濡れる＋絞る」の2ステップを前提としており不採択。
>
> ✅ 「乾けば乾くほど濡れるものは → タオル」
> 一ステップ、論理明快。

**② 頭で作るな、観察して作れ**
概念（秘密、平和）より物理現象や日常の事実から出発すること。AOF論文の生成物が失敗したのもこの理由。

**③ 逆説は二方向あること**
「食べるほど減るのに、食べないと増える → 食欲」は「食べる→減る」と「食べない→増える」の二方向。「暗いほどよく見える、明るいほど消えてしまう → 星」も同様。片方だけでは弱い。

**④ 答えを聞いた瞬間に世界が変わること**
答えを知る前と後で同じ言葉が別の意味に見える構造。「なるほど！」の納得感が必須。

### プロンプト制約

```
- 定番フォームは2〜3個まで
- よくある答え（影・時間・風・沈黙・炎）は禁止
- 一つの現象から一つの逆説で成立すること
- 答えが一意に定まること
- 日本語固有の要素を活用すること（音読み・訓読み・語呂・漢字構造）
- 英語から翻訳したものは不採択
```

## Pattern Taxonomy

日本語なぞなぞには英語にない固有パターンが存在する。特に文字分解系・逆読み系・漢字構造系はAOF論文が完全に見落としていた領域。

| パターン | 説明 | 例 |
|---|---|---|
| `paradox` | 逆説・現象系 | さむくなると伸びる → つらら |
| `char_extract` | 文字分解系 | 機械の真ん中 → 蚊（き**か**い） |
| `reverse_read` | 逆読み・回文系 | まるく → 車 |
| `pun` | 同音異義・ダジャレ系 | 食べられないパン → フライパン |
| `char_add_remove` | 文字足し引き系 | おしりに「う」をつけると少なくなる野菜 → なす |
| `kanji_structure` | 漢字構造系 | 漢字の部首・構造を分解 |
| `nazokake` | なぞかけ三段謎 | ○○とかけて△△と解く、その心は |

## Quality Criteria

- **一意性**：答えが一つに定まること（「屋根にはいるのに家にいないもの」は「ね」「や」が両方成立するため不採択）
- **一現象一逆説**：複数ステップの論理は不採択
- **観察ベース**：概念遊びより物理現象・日常観察から生まれたもの
- **ウェブ未検出**：検索サブエージェントによる重複確認済み
